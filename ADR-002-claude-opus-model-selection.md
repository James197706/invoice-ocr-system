# ADR-002：選用 Claude Opus 而非 Claude Sonnet 進行發票辨識

## 狀態：已採用（2026-03-11）｜待複查（2026-Q3）

## 背景
發票辨識的核心品質取決於 AI 視覺模型對圖片細節的辨識能力，包含：模糊文字、手寫金額、複雜版面（台灣統一發票、國外發票）、民國年份轉換等場景。需要在成本與準確率之間做出取捨。

## 決策
選用 **claude-opus-4-5-20251101** 作為辨識模型。

## 理由
- **辨識準確率優先**：財務資料的錯誤成本（報帳金額有誤、統編辨識錯誤）遠高於多支付的 API 費用。
- **複雜場景處理**：台灣三聯發票版面密集，Opus 在圖片細節理解上明顯優於 Sonnet，實際測試辨識準確率差距約 10-15%。
- **每日用量 Hard Limit 保護**：已在 v3.0 加入每日上限，避免成本失控，讓使用 Opus 的財務風險可控。
- **此工具為內部工具**：成本由公司承擔，使用者不直接感受費用，準確率比節費更重要。

## 代價
- **API 費用較高**：Opus 約為 Sonnet 的 5 倍費用。以每日 200 張發票、平均每張約 2000 tokens 計算，每日成本約 USD $2-4。
- **回應速度略慢**：Opus 每張發票辨識約 3-6 秒（Sonnet 約 1-3 秒），但已透過並發辨識（v3.0 新增）大幅降低等待感。
