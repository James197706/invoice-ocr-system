所有版本變更記錄。格式依循 [Keep a Changelog](https://keepachangelog.com/zh-TW/1.0.0/)。

## [3.2] — 2026-03-17

### 📘 平台內容更新
- **新增平台內使用說明頁籤**：將登入、上傳、審核、匯出與最佳實務直接放進平台，降低新同仁上手成本。
- **新增平台內版本更新頁籤**：使用者可在系統中直接查看當前版本與近期更新內容，不必回到程式碼或文件查找。

### ⚙️ 流程與穩定性優化
- **新增辨識前額度預檢**：若今日剩餘可用額度不足，系統會先限制可送出的張數並提示使用者，避免半途失敗。
- **修正預設欄位帶入邏輯**：預設部門、費用科目、專案代碼現在能正確補入空白辨識結果。
- **強化 AI JSON 解析**：兼容多段文字與 code fence 格式回應，降低解析失敗機率。
- **統一單檔上傳上限**：平台與後端設定一致改為 20MB，避免操作認知落差。

## [3.0] — 2026-03-12

### 🔒 資安強化
- **新增 API 每日 Hard Limit**：在程式碼層級設定每日辨識張數上限（預設 200 張，可透過 Secrets `DAILY_API_LIMIT` 調整）。超過上限直接擋住，防止帳單爆炸。側邊欄即時顯示今日用量進度條。
- **新增登入防暴力攻擊保護**：連續輸錯密碼 5 次後，Session 層級鎖定 60 秒，並顯示剩餘嘗試次數與倒計時提示。

### ⚡ 效能優化
- **並發辨識**：批次上傳多張發票時改用 `ThreadPoolExecutor` 並發送出 API 請求（最多 5 個同時），大幅縮短等待時間。原本辨識 10 張需序列等待 ~50 秒，現在約 ~10 秒。
- **Anthropic Client 快取**：改用 `@st.cache_resource` 快取 Client 實例，整個應用生命週期只建立一次連線，減少重複握手開銷。

### 🛠️ 程式碼品質（DRY 重構）
- **選項列表統一**：`DEPT_OPTIONS`、`ACCT_OPTIONS`、`TYPE_OPTIONS`、`PAY_OPTIONS`、`CURR_OPTIONS` 全數抽為模組頂層常數，消滅原本 sidebar 與 tab_review 各定義一份的重複問題。
- **版本號常數化**：新增 `APP_VERSION = "3.0"` 頂層常數，所有顯示版本號的地方統一引用，更新版本只需改一行。
- **錯誤處理細化**：`recognize_invoice()` 的 Exception 捕獲由籠統的 `except Exception` 拆分為明確的 `json.JSONDecodeError`、`KeyError`、`AuthenticationError`、`RateLimitError`，各自提供對用戶有意義的錯誤訊息。

### 🎨 UX 體驗改善
- **刪除二次確認**：「🗑️ 刪除此筆」現在會先顯示確認對話（含廠商名稱），明確告知「刪除後無法復原」，再由用戶確認或取消，防止誤觸。
- **清除全部二次確認**：側邊欄「🗑️ 清除」同樣加入二次確認步驟。
- **用量警告提示**：今日用量達 90% 時，側邊欄顯示黃色警告提示。

## [2.0] — 2026-03-11（首次雲端部署）

### 新增
- 雲端版完整功能：登入保護、三頁籤介面（上傳辨識、審核資料、匯出 Excel）。
- `st.secrets` 管理 API Key 與密碼，不再需要使用者手動輸入 API Key。
- 支援台灣統一發票（二聯、三聯）、一般收據、國外發票、電子發票。
- 3 個 Excel 工作表：發票彙總、品項明細、統計分析（費用科目、部門、廠商前10、付款方式）。
- 側邊欄即時統計：已辨識張數、合計金額。
- PDF 上傳支援（PyMuPDF 轉圖片後辨識）。
